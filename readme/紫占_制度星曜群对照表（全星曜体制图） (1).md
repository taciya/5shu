# 紫占三层卦 × 追四化 × 飞四化 × 化解模块

## 一、使用目的
用于**教学 / 实战统一流程**：
> 从「表象问题」→「星曜/宫位故障」→「四化定位」→「系统级化解方案」

---

## 二、标准流程总览（主流程）

【步骤 0｜问题输入】
- 问卦占事 / 占命 / 占状态
- 明确：
  - 是「能力问题」？
  - 还是「节律 / 制度 / 情绪 / 结构问题」？

⬇️

【步骤 1｜锁定三层卦】
- 主卦：事件**发生场域**（外在表现）
- 十分卦：问题**结构来源**（中层机制）
- 分钟卦：**核心执行系统**（当下运转状态）

⬇️

【步骤 2｜识别“异常星曜 / 宫位”】
检查以下任一条件即视为“系统异常”：
- 星曜：
  - 化忌 / 被追忌 / 高比例飞忌
  - 陷 / 不 + 被激活
- 宫位：
  - 命 / 官禄 / 迁移 / 疾厄 被反复触发

⬇️

【步骤 3｜追四化：定位“问题源头”】【原因定位】
- 追禄：
  - 事件为何而起？
  - 哪个系统最初被放大？
- 追忌：
  - 哪个环节开始失效？
- 追权 / 追科：
  - 是否存在人为 / 制度性干预？

⚠️ 注意：
> 追四化只回答一句话：
> **「问题最初是从哪里开始的？」**

⬇️

【步骤 4｜飞四化：观察“能量扩散”】【后果分析】
- 每一层卦分别检查：
  - 飞禄 → 被放大的系统
  - 飞权 → 被卡住 / 强制的系统
  - 飞科 → 被延缓 / 修补的系统
  - 飞忌 → 实际崩溃 / 终止点

重点判断：
- 飞忌是否形成：
  - 向心（回到命 / 疾厄）
  - 离心（外推至官禄 / 迁移 / 交友）

⬇️

【步骤 5｜系统判定：这是“能力问题”还是“接口问题”】【关键分流】

- 若：
  - 命宫主星本质良好（庙 / 旺）
  - 但被飞忌 / 追忌打断
  → 判定为：
  **「接口 / 调度 / 合法性问题」**

- 若：
  - 主星本身陷 + 多重忌
  → 判定为：
  **「核心模块老化 / 不适配」**

⬇️

【步骤 6｜调用“化解模块”】【系统修复】

### 常用化解调用表（速查）

- 天机宕机 → 天相 × 疾厄 / 父母
- 太阴耗竭 → 天梁 × 福德
- 巨门是非 → 天相 × 官禄 / 文昌
- 廉贞失序 → 紫微 × 父母
- 七杀失控 → 天梁 × 疾厄
- 贪狼迷失 → 紫微 × 父母
- 破军反复 → 天府 × 田宅

⬇️

【步骤 7｜最终结论输出】
- 用一句话总结：
  > 不是「好 / 坏」，而是：
  > **哪个系统现在不该由它来扛**

- 给出：
  - 当前状态解释
  - 可行的“系统级调整方向”

---

## 三、教学用提醒（非常重要）

1️⃣ 不要急着解释星性
→ 先找「系统断点」

2️⃣ 四化不是吉凶
→ 是**系统状态变化信号**

3️⃣ 所谓化解
→ 不是补运，而是**换模块 / 换接口 / 换节律**
