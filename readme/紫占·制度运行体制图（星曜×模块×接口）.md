# 紫占·制度运行体制图（星曜×模块×接口）

> 本图用于说明：**星曜不是性格标签，而是系统模块**。  
> 当一个人“卡住、宕机、混乱”，本质是**模块失职或接口失配**。

---

## 一、紫占整体制度分层（总览）

```
【中枢层】        紫微 · 天府
   ↓ 决策 / 定调
【制度层】        天相 · 天梁
   ↓ 合法 / 修复
【执行层】        武曲 · 七杀 · 破军
   ↓ 行动 / 变现
【认知层】        天机 · 巨门 · 文昌 · 文曲
   ↓ 运算 / 定义 / 编码
【调欲层】        贪狼 · 太阴 · 太阳 · 天同
   ↓ 能量 / 动机 / 情绪
```

---

## 二、关键制度模块说明（核心）

### 1️⃣ 紫微｜最高决策中枢
- 功能：定调、授权、方向选择
- 失效表现：群龙无首、反复试错
- 只能被：**天府（资源）或天相（制度）**承接

---

### 2️⃣ 天府｜资源与承载系统
- 功能：库存、稳定、长期维持
- 失效表现：有方向但撑不久
- 常与紫微组成「稳定政权」

---

### 3️⃣ 天相｜制度与合法性接口 ⭐
- 功能：规则、流程、认证、许可
- 关键点：**让个人行为被系统承认**
- 能修复：
  - 天机宕机（给规则）
  - 巨门争议（给判准）

---

### 4️⃣ 天梁｜制度修复与兜底
- 功能：善后、补漏、延寿
- 特征：
  - 不提升效率
  - 但防止系统崩溃

---

## 三、认知与执行模块（你关心的重点）

### 5️⃣ 天机｜计算 / 推演 / 调度
- 正常：快速、多线程、灵活
- 宕机：过载、被打断、节律错乱
- **必须接天相 or 天府，才能“安稳化”**

---

### 6️⃣ 巨门｜定义权 / 审判接口
- 功能：提出问题、制造争议
- 不是坏，而是：
  - 没有它，系统无法修正
- 与天相关系：
  - 巨门 = 提出案件
  - 天相 = 给出判准

---

### 7️⃣ 武曲｜执行与量化
- 功能：落实、产出、现金流
- 失效：努力无回报
- 常与七杀、破军组成执行组

---

### 8️⃣ 七杀｜突破与强制
- 功能：打破旧结构
- 风险：若无天相，易违法/失序

---

### 9️⃣ 破军｜重启与重构
- 功能：推倒重来
- 风险：反复归零

---

## 四、辅助系统（记录与能量）

### 🔹 文昌 · 文曲
- 功能：记录、表达、规则编码
- 是制度能否“被复制”的关键

### 🔹 贪狼 / 太阴 / 太阳 / 天同
- 功能：欲望、情绪、动机
- 问题多出在：
  - 没被制度层接住

---

## 五、典型系统修复路径（示例）

### ❌ 天机宕机（你前面的问题）
```
天机过载
 → 巨门争议放大
 → 铃星噪音
 → 系统停机
```

### ✅ 正确修复
```
天相介入（疾厄 / 父母）
 → 给规则 / 给节律
 → 天府承载
 → 天机恢复稳定运算
```

---

## 六、一句话总结（体系核心）

> **紫占不是算命，是系统工程。**  
> 星曜不是好坏，是模块；  
> 四化不是运气，是状态变化；  
> 解盘不是安慰，是找对“该谁接手”。
