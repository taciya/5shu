以下为【紫占体系｜系统故障 × 化解模块】标准对照表。
本表用于回答：
👉「某星曜/宫位发生系统性问题时，是否真的有解？解从哪里来？在什么条件下才成立？」

————————————————————
一、核心原则（务必先读）
1️⃣ 化解 ≠ 抵消
→ 紫占中的“化解”是【系统接管 / 职能转移】，不是把问题抹掉。

2️⃣ 只有“高阶系统”才能化解“低阶故障”
→ 同级星曜只能对冲，不能稳定。

3️⃣ 化解必须落在【接口宫位】
→ 星曜对了，宫位不对 = 无效安抚。

————————————————————
二、常见系统故障 × 可接手模块对照

【1】天机宕机 / 停机型天机
- 故障定义：
  推演中断、学习跟不上、思维忽快忽慢，对外界刺激高度敏感。

- 允许接手的星曜：
  ✔ 天相（首选）
  ✔ 天梁（次选）

- 有效落宫：
  ◎ 疾厄宫（系统底层规则）
  ◎ 父母宫（上位许可 / 规范来源）

- 机制说明：
  天相负责“固定规则”，让天机不必反复演算；
  天梁负责“兜底逻辑”，让系统允许慢速运行。

- ❌ 无效方式：
  文昌/文曲补课、贪狼刺激、破军强推

————————————————————
【2】巨门黑箱 / 定义权卡死
- 故障定义：
  说不清、被误解、是非缠身，越解释越乱。

- 允许接手的星曜：
  ✔ 天相（制度化）
  ✔ 紫微（定调）

- 有效落宫：
  ◎ 官禄宫（公共规则场）
  ◎ 父母宫（权威背书）

- 机制说明：
  巨门制造问题，天相把问题送进流程；
  紫微一旦介入，争议直接“终审”。

————————————————————
【3】廉贞失序 / 规则反噬
- 故障定义：
  明知不合规仍想试探，或被制度反复处罚。

- 允许接手的星曜：
  ✔ 天梁（伦理边界）
  ✔ 天府（资源缓冲）

- 有效落宫：
  ◎ 福德宫（价值系统）
  ◎ 田宅宫（长期结构）

————————————————————
【4】贪狼成瘾 / 动机失控
- 故障定义：
  欲望驱动强烈但不可持续，学一阵就崩。

- 允许接手的星曜：
  ✔ 天梁（节制）
  ✔ 天相（合法化路径）

- 有效落宫：
  ◎ 疾厄宫（节律）
  ◎ 官禄宫（正当出口）

————————————————————
【5】破军过载 / 重启失败
- 故障定义：
  反复推翻重来，人生频繁清零。

- 允许接手的星曜：
  ✔ 天府（存量系统）
  ✔ 紫微（主权重建）

- 有效落宫：
  ◎ 田宅宫（基础结构）
  ◎ 命宫（核心权限）

————————————————————
三、快速判断口诀（教学用）
- 算力问题 → 看天机有没有被迫停机
- 争议问题 → 看巨门有没有送进天相
- 失控问题 → 看贪狼有没有梁/相接手
- 重启失败 → 看有没有府/紫接管

————————————————————
四、重要结论
👉 真正的“转运”，从来不是加一颗吉星，
👉 而是【把问题交给它该去的系统部门】。
